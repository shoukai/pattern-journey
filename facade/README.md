# 外观模式介绍

### 面向对象的一个比较重要的法则：

迪米特法则（最少知识原则）：一个软件实体应当尽可能少的与其他实体发生相互作用。

### 外观模式：

为子系统中的一组接口提供一个一致的界面，此模式定义了一个高层接口。这个接口使得这一子系统更加容易使用。说白了，就可以理解为封装

### 外观模式的核心：

为子系统提供统一的入口，封装子系统的复杂性，便于客户端调用
